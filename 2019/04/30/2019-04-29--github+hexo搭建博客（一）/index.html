2019.4.29 GitHub+hexo博客搭建（一）

Created at 2019-04-29 Updated at 2019-04-30 Category Blog Tag Blog

一、开通 GitHub page

登陆GitHub，点击右上角“+”，点击New respository创建一个新仓库，

在新的页面中，将 Repository name（仓库的名称）设置为 username.github.io ，其中 username 是你的用户名,这里VAnSTerren是我的用户名，然后点击页面最下方的”Create repository” ，创建这个仓库。

二、下载安装配置Git

  • Hexo 是一个基于 Node.js 的静态站点生成器，但如果要将站点 publish 到 Github 上的话，还需要依赖 Git

2.1 Git 下载（Windows）： https://git-scm.com/download/win


下载完成后，点击安装一路next即可。

2.2 配置Git

  • 需要先生成一个ssh key 密钥，我们本地的 Git 和远端的 Github 之间通过 SSH 加密传输数据。

在桌面右键，点击 “Git Bash Here” 打开 bash ：


输入 cd ~ 进入用户主目录

接下来执行 ls -al | grep .ssh 检查是否存在一个名为 “.ssh” 的文件夹。

如果显示有记录，即存在 “.ssh” 文件夹的话。再执行命令 ls .ssh -al 检查一下该文件夹下是否存在两个分别叫做 “id_rsa” 和 “id_rsa.pub” 的文件。如下图所示

如果输出如图包含 “id_rsa” 和 “id_rsa.pub” ，可以跳过下一步，如果不存在 “.ssh” 文件夹，或者 “.ssh” 文件夹下不存在 “id_rsa” 和 “id_rsa.pub” 密钥对文件，即需要手动创建它们。
执行 ssh-keygen -t rsa -C “user@server.com“ 生成密钥对，其中 -t rsa 指定了密钥的类型为 RSA ， -C “user@server.com“ 为密钥添加一段注释，可以填写为你注册 Github 时所使用的邮箱。
生成密钥对的过程中需要填写三个参数，这里直接跳过，使用默认值

此时再执行 ls .ssh -al 就可以看见我们想要的文件已经生成。
其中 “id_rsa” 为私钥， “id_rsa.pub” 为公钥，我们需要将公钥导入进你的 Github 账户中。

执行 cat ./.ssh/id_rsa.pub ，然后将输出（也就是公钥）复制进剪切板中（选取输出的全部内容，Ctrl + Insert ）：

回到 Github ，点击右上角的头像，再点击 “Settings” 进入设置页面：

进入设置页面后，点击 “Personal settings” 下的 “SSH and GPG keys” ，再点击右侧的 “New SSH key”：

接下来输入 Title 和 Key 的内容，Title 可以按照自己的喜好来填，比方说 “github page “或者其他都可以
输入完 Title 之后，将刚才复制进剪切板的公钥粘贴在 Key 中，然后点击下面的 “Add SSH key” 按钮。

接下来 Github 会要求输入密码以继续操作，这个密码就是你的 Github 账户的密码。
完成后，会回到刚才的 “SSH and GPG keys” 页面，如果成功的话会看见页面上多出了一个新的 SSH key;

至此，公钥导入成功。接下来回到 Git Bash ，验证一下能否正常连接。
执行 ssh -T git@github.com ，当出现 “(yes/no)” 时输入 “yes” 同意，正常情况下会出现 “…You‘ve successfully authenticated…” ，表示连接可以正常建立:

在配置完密钥对之后，我们还需要设置一下 Git 的用户名和邮箱。
执行 git config –global user.name “username” ，这里 “username” 为 Github 账户名。
执行 git config –global user.email “user@server.com“ ，这里”user@server.com“ 填写为注册 Github 时使用的邮箱。
以上两个命令在参数和执行正确的情况下都不会产生输出。
2.3 测试
最后，测试一下能否正常向我们的 Github Page 上传页面。
执行 git clone https://github.com/username/username.github.io ，其中两个 “username” 均替换为你的 Github 账户名，

中途可能会弹出框输入自己的GitHub账号密码，在新弹出的窗口输入密码
上述步骤完成后，在浏览器中再次打开你的 Github Page（这里以我的 VAnSterren.github.io 为例），就可以看见 Hello World 了

三、安装 Node.js 和 Hexo

###3.1 安装Node.js

  • Hexo 基于 Node.js

下载 Node.js 的安装包： http://nodejs.cn/download/
下载完成后，进行安装，一路next即可。
然后打开命令提示符，依次执行 node -v 和 npm -v ，正常情况下这两条命令会输出 Node.js 和 npm 包管理器的版本号。

    五、文章管理

文章的 Markdown 文件存放在博客文件夹下的 source 目录下。

默认情况下，新建的文章都会存放在 “_post” 子目录下，现在里面只有一个叫 “hello-world.md” 的文件，这就是刚刚 publish 到 Github 上的博客中的唯一一篇文章的 Markdown 源文件。

5.1 新建文章

回到 cmd ，在博客文件夹的根目录下执行 hexo new 来向 post 中添加一篇指定标题的空白文章，例如 hexo new “my first article” ：

在 source/_post 下会生成一个与之对应的新 Markdown 文件，这就是我们的新文章，选择你喜欢的文本编辑器对它进行编辑即可

编辑并保存之后，在 cmd 中依次执行
hexo clean
hexo g
hexo d
即可将新文章部署至 Github

5.2 删除文章

只需要在执行 hexo clean 前删掉对应的 Markdown 文件即可
